## Overview

HotCRM's Service Cloud provides comprehensive customer support capabilities including multi-channel case management, SLA tracking with breach prediction, intelligent routing, and RAG-enabled knowledge base.

**Package:** `@hotcrm/support`  
**Objects:** 21 | **AI Actions:** 3 | **Hooks:** 6

## Case Management

### Multi-Channel Support

**Object:** `case.object.ts`

Support 7 communication channels from a single interface:

```typescript
// Channels
{
  channels: [
    'Email',           // Traditional email
    'Web',             // Web forms
    'Phone',           // Call center
    'WeChat',          // Chinese social platform
    'Chat',            // Live chat
    'Mobile',          // Mobile app
    'Walk-in',         // In-person support
  ],
  
  fields: [
    'case_number',       // Auto-generated
    'subject',
    'description',
    'account_id',
    'contact_id',
    'status',            // New, Working, Escalated, Resolved, Closed
    'priority',          // Low, Medium, High, Critical
    'origin',            // Channel
    'type',              // Question, Problem, Feature Request
    'reason',            // Categorization
    'owner_id',          // Agent assigned
    'sla_policy_id',     // SLA rules
    'csat_score',        // Customer satisfaction (1-5)
  ]
}
```

**AI Features:**
- **Intelligent Routing**: Auto-assign to best agent based on skills, availability, workload
- **Solution Recommendations**: Suggest knowledge articles
- **Sentiment Analysis**: Detect customer frustration
- **Priority Prediction**: Auto-escalate critical cases

## SLA Management

### SLA Templates

**Object:** `sla_template.object.ts`

Define response and resolution targets:

```typescript
{
  fields: [
    'name',
    'priority',            // Critical, High, Medium, Low
    'first_response_hours', // Target for first response
    'resolution_hours',    // Target for resolution
    'business_hours_only', // Boolean
  ]
}
```

**Example SLA Targets:**

| Priority | First Response | Resolution |
|----------|---------------|------------|
| Critical | 1 hour | 4 hours |
| High | 4 hours | 24 hours |
| Medium | 8 hours | 3 days |
| Low | 24 hours | 7 days |

### SLA Policies

**Object:** `sla_policy.object.ts`

Multi-tier support with escalation:

```typescript
{
  fields: [
    'name',
    'tier',                // Platinum, Gold, Silver, Bronze, Standard
    'is_active',
    'account_criteria',    // JSON filter
    'sla_template_id',
    'escalation_enabled',
    'escalation_hours',    // Hours before escalation
    'entitlement_process', // Premium vs. Standard

}
```

**Support Tiers:**
- **Platinum**: 24/7, 1hr response, dedicated CSM
- **Gold**: 24/5, 4hr response, priority queue
- **Silver**: 12/5, 8hr response
- **Bronze**: 8/5, 24hr response
- **Standard**: Best effort

### Routing Rules

**Object:** `routing_rule.object.ts`

Define intelligent assignment logic:

```typescript
{
  fields: [
    'name',
    'is_active',
    'priority',            // Rule precedence
    'case_criteria',       // JSON filter (type, priority, origin)
    'assignment_method',   // Round Robin, Least Busy, Skill Based, AI Powered
    'skill_requirements',  // Required skills
    'team_id',             // Target team
  ]
}
```

**Assignment Methods:**
1. **Round Robin**: Distribute evenly
2. **Least Busy**: Assign to agent with fewest open cases
3. **Skill Based**: Match required skills
4. **AI Powered**: ML-based optimal assignment

## Escalation Management

### Escalation Rules

**Object:** `escalation_rule.object.ts`

Automated escalation workflows:

```typescript
{
  fields: [
    'name',
    'is_active',
    'case_criteria',       // When to escalate
    'escalation_trigger',  // SLA Violation, No Response, Priority Change
    'escalation_actions',  // JSON array of actions
    'notify_users',        // Users to notify
    'reassign_to',         // New owner
  ]
}
```

**Escalation Triggers:**
- SLA violation imminent
- No agent response in X hours
- Customer frustration detected (sentiment)
- Case reopened multiple times

**Escalation Actions:**
- Email notifications
- Slack/Teams alerts
- Priority increase
- Reassignment to senior agent
- Manager involvement

## Analytics & Reporting

### Key Metrics

- **Case Volume**: Cases by channel, type, priority
- **SLA Compliance**: % of cases meeting SLA
- **First Contact Resolution (FCR)**: % resolved on first contact
- **Average Handle Time (AHT)**: Time to resolve
- **Customer Satisfaction (CSAT)**: Average CSAT score
- **Agent Productivity**: Cases per agent per day
- **Backlog**: Open cases aging report

### AI-Powered Insights

```typescript
// Case volume forecasting
POST /api/actions/forecast_case_volume
{
  "time_period": "next_30_days"
}

// Agent capacity planning
POST /api/actions/agent_capacity
{
  "team_id": "team_123"
}

// Knowledge gap analysis
POST /api/actions/knowledge_gaps
// Returns: Topics with many cases but no articles
```

## API Examples

### Create a Case

```typescript
POST /api/objects/case
{
  "subject": "Password reset issue",
  "description": "Customer cannot reset password",
  "account_id": "acc_123",
  "contact_id": "con_456",
  "origin": "Email",
  "priority": "High"
}
```

### AI Routing

```typescript
POST /api/actions/intelligent_routing
{
  "case_id": "case_789"
}
// Returns: Recommended agent + Reasoning
```

### Search Knowledge

```typescript
POST /api/actions/search_knowledge
{
  "query": "How to configure SSO?",
  "max_results": 5
}
// Returns: Top 5 articles with similarity scores
```
